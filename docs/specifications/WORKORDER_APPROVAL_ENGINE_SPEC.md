## 需求与背景

### 业务场景

企业内部存在大量需要多级审批的工作流程（如采购申请、费用报销、项目立项、合同审批等），传统通过邮件或纸质传递的方式存在以下痛点：

| 痛点 | 现状描述 |
|------|----------|
| 流程不可见 | 申请人无法实时知晓审批进度 |
| 审批随意性 | 缺乏标准化流转规则，审批人可跳过步骤 |
| 责任不清 | 审批记录散落于邮件，无统一归档 |
| 效率低下 | 跨部门审批需人工逐级传递 |

### 核心价值

```
┌─────────────────────────────────────────────────────────────┐
│                    工单审批流程引擎                          │
├─────────────────────────────────────────────────────────────┤
│  状态机驱动    │  引擎内核管理状态转换，规则外操作自动拦截    │
├────────────────┼────────────────────────────────────────────┤
│  多级审批链    │  支持并行/串行审批，支持条件分支            │
├────────────────┼────────────────────────────────────────────┤
│  实时通知      │  审批状态变更即时推送，支持邮件/WebSocket   │
├────────────────┼────────────────────────────────────────────┤
│  完整审计      │  所有操作留痕，支持追溯与合规审查           │
└─────────────────────────────────────────────────────────────┘
```

## 当前 Phase 对应实施目标

### Phase 1: 核心链路交付

**范围定义**: 覆盖工单从创建到最终审批完成的全路径，但限制为**单工单类型、串行审批链、无条件分支**。

```
Phase 1 交付物边界
──────────────────────────────────────────────────────────────

  ┌────────┐    ┌─────────┐    ┌──────────┐    ┌──────────┐
  │ 创建   │───▶│ 待审批  │───▶│ 审批中   │───▶│ 已完成  │
  │ 工单   │    │ (一级)  │    │ (二级)   │    │          │
  └────────┘    └─────────┘    └──────────┘    └──────────┘
                      │               │
                      ▼               ▼
               ┌───────────┐   ┌───────────┐
               │  审批拒绝  │   │  审批拒绝  │
               └───────────┘   └───────────┘

  通知触发点: [工单创建] [一级审批完成] [二级审批完成/拒绝]
```

## 状态机定义

### 状态枚举

```typescript
enum WorkOrderStatus {
  DRAFT = 'DRAFT',           // 草稿状态（可选）
  PENDING = 'PENDING',       // 待一级审批
  APPROVED_L1 = 'APPROVED_L1', // 一级审批通过，待二级审批
  APPROVED_L2 = 'APPROVED_L2', // 二级审批通过
  REJECTED = 'REJECTED',     // 审批拒绝
  COMPLETED = 'COMPLETED',  // 流程完成
  CANCELLED = 'CANCELLED'   // 已取消
}
```

### 事件枚举

```typescript
enum ApprovalEvent {
  SUBMIT = 'SUBMIT',           // 提交工单
  APPROVE_L1 = 'APPROVE_L1',   // 一级审批通过
  APPROVE_L2 = 'APPROVE_L2',   // 二级审批通过
  REJECT_L1 = 'REJECT_L1',     // 一级拒绝
  REJECT_L2 = 'REJECT_L2',     // 二级拒绝
  CANCEL = 'CANCEL'            // 取消工单
}
```

### 状态转换规则

| 当前状态 | 事件 | 目标状态 | 守卫条件 |
|----------|------|----------|----------|
| DRAFT | SUBMIT | PENDING | 必填字段完整 |
| PENDING | APPROVE_L1 | APPROVED_L1 | 审批人为一级审批角色 |
| PENDING | REJECT_L1 | REJECTED | 审批人为一级审批角色 |
| APPROVED_L1 | APPROVE_L2 | COMPLETED | 审批人为二级审批角色 |
| APPROVED_L1 | REJECT_L2 | REJECTED | 审批人为二级审批角色 |
| * | CANCEL | CANCELLED | 仅创建人或管理员可取消 |

## API 接口设计

### 工单管理

| 接口 | 方法 | 描述 |
|------|------|------|
| `/api/workorders` | POST | 创建工单 |
| `/api/workorders/{id}` | GET | 获取工单详情 |
| `/api/workorders` | GET | 查询工单列表 |
| `/api/workorders/{id}/approve` | POST | 审批通过 |
| `/api/workorders/{id}/reject` | POST | 审批拒绝 |
| `/api/workorders/{id}/history` | GET | 获取审批历史 |

## 技术约束

- **审批链固定**: 仅支持预定义的串行二级审批
- **状态不可逆**: 审批通过后不可撤回
- **单工单类型**: 仅支持"通用审批"类型
