## 1. 需求与背景

### 1.1 业务场景

工单审批流程是数字化资产管理系统的核心模块。用户（审批人）通过前端审批页面完成工单审批操作（通过/拒绝/驳回），后端驱动状态机状态流转并触发对应的通知机制（站内信/邮件/短信）。

### 1.2 当前痛点

| 痛点 | 描述 |
|------|------|
| **操作割裂** | 审批操作需跳转到独立管理系统，用户体验不连贯 |
| **状态硬编码** | 状态变更依赖硬编码逻辑，缺乏统一状态机管理 |
| **通知遗漏** | 审批结果通知被动触发，存在时延或遗漏风险 |

### 1.3 核心目标

| 目标 | 说明 |
|------|------|
| **G1** | 建立统一的状态机驱动审批引擎，实现前端审批页面与后端状态流转闭环 |
| **G2** | 确保状态变更后即时触发多渠道通知机制 |
| **G3** | 审批操作具备幂等性，防止重复审批 |
| **G4** | 审批流程可审计，支持完整的审批历史追溯 |

## 3. 边界约束

### 3.1 功能边界

| 约束项 | 约束内容 | 优先级 |
|--------|----------|--------|
| **审批操作类型** | 支持通过（`APPROVED`）、拒绝（`REJECTED`）、驳回（`REVISION_REQUIRED`）三种操作 | P0 |
| **状态机定义** | 必须遵循：`PENDING` → `APPROVED` / `REJECTED` / `REVISION_REQUIRED`，状态不可逆 | P0 |
| **审批权限** | 仅工单指定审批人或审批角色组可执行审批操作 | P0 |
| **并发控制** | 同工单并发审批操作必须加分布式锁，防止重复审批 | P0 |
| **通知触发时机** | 状态变更成功后**同步**触发通知事件（异步队列处理） | P1 |
| **前端协议** | 前后端通过 RESTful API 交互，数据格式为 JSON | P1 |
| **幂等性** | 同一工单重复审批操作返回 409 Conflict，不修改状态 | P0 |

### 3.2 技术边界

| 约束项 | 约束内容 | 实施建议 |
|--------|----------|----------|
| **状态机实现** | 后端采用状态模式（State Pattern）或规则引擎实现，禁止硬编码 if-else 分支 | 使用 `transitions` 库或自定义状态机类 |
| **事务要求** | 审批状态写入与事件发布必须在同一事务内完成（Outbox Pattern 或本地消息表） | 避免分布式事务 |
| **通知幂等性** | 通知发送必须具备幂等性（基于 `notification_id`），防止重复推送 | 使用唯一消息 ID |
| **API 响应时间** | 单次审批操作 P99 ≤ 500ms | 避免长事务 |
| **错误处理** | 所有业务异常必须抛出标准异常类型，禁止裸 raise | 统一异常处理 |

### 3.3 数据边界

| 约束项 | 约束内容 | 字段规范 |
|--------|----------|----------|
| **工单状态字段** | `status`（PENDING/APPROVED/REJECTED/REVISION_REQUIRED）、`updated_at`、`updated_by` | 必须包含时间戳和操作人 |
| **审批记录字段** | `approval_id`、`work_order_id`、`action`、`approver_id`、`comment`、`created_at` | 主键采用 UUID |
| **通知记录字段** | `notification_id`、`recipient_id`、`channel`、`template_id`、`status`、`sent_at` | 支持多渠道 |

## 6. `backend/models/approval_record.py` 实现规范

### 6.1 类结构定义

```python
"""
工单审批记录数据模型

该模块定义了工单审批流程中的审批记录数据结构，
包括审批动作类型、审批状态以及相关的时间戳和审计字段。

Author: 待填充
Version: 1.0.0
"""

from dataclasses import dataclass, field
from datetime import datetime
from enum import Enum
from typing import Optional
from uuid import UUID, uuid4


class ApprovalAction(str, Enum):
    """
    审批动作类型枚举
    
    定义了工单审批流程中所有可能的审批操作类型。
    状态流转规则：
        - PENDING: 初始状态，可流转至 APPROVED / REJECTED / REVISION_REQUIRED
        - APPROVED: 最终状态，不可再流转
        - REJECTED: 最终状态，不可再流转
        - REVISION_REQUIRED: 中间状态，可再次提交审批
    """
    APPROVED = "APPROVED"
    REJECTED = "REJECTED"
    REVISION_REQUIRED = "REVISION_REQUIRED"


class ApprovalStatus(str, Enum):
    """
    审批记录状态枚举
    
    用于标识审批记录本身的处理状态。
    """
    PENDING = "PENDING"
    COMPLETED = "COMPLETED"
    FAILED = "FAILED"
    CANCELLED = "CANCELLED"


@dataclass
class ApprovalRecord:
    """
    工单审批记录数据类
    
    Attributes:
        id: 审批记录唯一标识符（UUID）
        work_order_id: 关联工单的唯一标识符
        action: 审批动作类型
        approver_id: 审批人唯一标识符
        approver_name: 审批人姓名
        comment: 审批意见/备注
        status: 审批记录状态
        created_at: 审批记录创建时间
        updated_at: 审批记录最后更新时间
    
    Example:
        >>> record = ApprovalRecord(
        ...     work_order_id=uuid4(),
        ...     action=ApprovalAction.APPROVED,
        ...     approver_id=uuid4(),
        ...     approver_name="张三",
        ...     comment="审批通过"
        ... )
        >>> record.id
        UUID('...')
    """
    
    work_order_id: UUID
    action: ApprovalAction
    approver_id: UUID
    approver_name: str
    comment: Optional[str] = None
    status: ApprovalStatus = ApprovalStatus.PENDING
    id: UUID = field(default_factory=uuid4)
    created_at: datetime = field(default_factory=datetime.utcnow)
    updated_at: datetime = field(default_factory=datetime.utcnow)
    
    def __post_init__(self) -> None:
        """
        数据类初始化后验证
        
        Raises:
            ValueError: 当必填字段为空或格式不正确时抛出
        """
        if not self.work_order_id:
            raise ValueError("work_order_id 不能为空")
        if not self.approver_id:
            raise ValueError("approver_id 不能为空")
        if not self.approver_name:
            raise ValueError("approver_name 不能为空")
        if not isinstance(self.action, ApprovalAction):
            raise ValueError(f"action 必须是 ApprovalAction 枚举类型，实际值: {type(self.action)}")
    
    def mark_completed(self) -> None:
        """
        标记审批记录为完成状态
        
        该方法用于在审批流程成功完成后调用，
        更新审批记录状态并刷新更新时间戳。
        
        Example:
            >>> record.mark_completed()
            >>> record.status
            <ApprovalStatus.COMPLETED: 'COMPLETED'>
        """
        self.status = ApprovalStatus.COMPLETED
        self.updated_at = datetime.utcnow()
    
    def mark_failed(self) -> None:
        """
        标记审批记录为失败状态
        
        该方法用于在审批流程执行失败时调用，
        更新审批记录状态并刷新更新时间戳。
        """
        self.status = ApprovalStatus.FAILED
        self.updated_at = datetime.utcnow()
    
    def to_dict(self) -> dict:
        """
        转换为字典格式
        
        Returns:
            dict: 包含所有字段的字典表示
        """
        return {
            "id": str(self.id),
            "work_order_id": str(self.work_order_id),
            "action": self.action.value,
            "approver_id": str(self.approver_id),
            "approver_name": self.approver_name,
            "comment": self.comment,
            "status": self.status.value,
            "created_at": self.created_at.isoformat(),
            "updated_at": self.updated_at.isoformat(),
        }
```
