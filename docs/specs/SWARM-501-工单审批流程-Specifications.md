# SWARM-501 工单审批流程 - Specifications

## 1. 需求与背景

### 1.1 业务目标

实现工单审批流程的前端交互与后端状态机联动，用户可在工单详情页提交审批意见并触发工单状态流转。

### 1.2 现状描述

- 工单系统基础结构已存在（工单 CRUD、工单详情页）
- 后端状态机框架已搭建（基于内部状态机框架）
- 审批节点定义已配置（审批人角色、审批条件）
- 多层服务架构已就绪：
  - `ApprovalService`：审批核心服务
  - `ApprovalChainService`：审批链服务
  - `NotificationService`：通知服务
  - `RetirementService`：退休/报废服务

### 1.3 目标描述

| 目标项 | 说明 |
|--------|------|
| 前端审批入口 | 工单详情页增加审批意见提交区域 |
| 状态机联动 | 审批提交后触发后端状态机状态流转 |
| 状态同步回显 | 状态流转结果同步回显至前端页面 |
| 审批链路日志 | 完整的审批链路日志记录 |
| 消息通知 | 审批完成后触发消息队列通知相关方 |

## 3. 边界约束

### 3.1 功能边界

| 约束项 | 说明 | 优先级 |
|--------|------|--------|
| 审批角色 | 仅限拥有审批权限的角色可见审批入口 | 🔴 High |
| 审批状态 | 仅 `待审批` 状态的工单可提交审批 | 🔴 High |
| 意见必填 | 审批意见字段为必填，最小 5 字符 | 🔴 High |
| 审批类型 | 支持 `通过`、`拒绝`、`退回修改` 三种操作 | 🔴 High |
| 幂等性 | 同一工单重复提交审批需防重处理 | 🟡 Medium |
| 乐观锁 | 使用 version 字段进行并发控制 | 🟡 Medium |

### 3.2 技术边界

| 约束项 | 说明 |
|--------|------|
| 前端框架 | React 18 + TypeScript + Vite |
| 后端框架 | Python FastAPI / Spring Boot (双轨) |
| 状态机 | 内部状态机框架 (`approval_state_machine.py`) |
| API 风格 | RESTful + JSON |
| 消息队列 | Redis Publisher → NotificationConsumer |

### 3.3 非功能边界

| 约束项 | 说明 |
|--------|------|
| 响应时限 | 审批提交 API P99 ≤ 500ms |
| 错误恢复 | 状态机流转失败需事务回滚，前端提示重试 |
| 日志审计 | 所有审批操作写入 `approval_record` / `audit_log` 表 |

### 3.4 代码质量边界

| 约束项 | 说明 | 对应 AC |
|--------|------|---------|
| AST 静态检查 | 代码变更不引入新的语法错误 | AC-003 |
| Docstring 规范 | 所有修改的函数包含文档注释 | AC-004 |
| Import 正常 | 变更后的模块可被正常 import | AC-005 |

### 5.4 注意事项

1. **禁止跨层开发**：必须严格遵循层级序列，禁止在前端组件开发时直接调用数据库
2. **测试先行**：每个层级开发前需先编写测试用例
3. **向后兼容**：数据库字段变更需保持向后兼容（新增字段 nullable）
4. **幂等处理**：审批提交接口需实现幂等性，防止重复提交
5. **乐观锁**：使用 version 字段进行并发控制，避免状态冲突

## 7. 附录

### 7.1 审批状态枚举

| 状态码 | 状态名 | 说明 |
|--------|--------|------|
| `PENDING` | 待提交 | 工单已创建，待提交审批 |
| `PENDING_APPROVAL` | 待审批 | 已提交，等待审批 |
| `IN_APPROVAL` | 审批中 | 正在审批流程中 |
| `APPROVED` | 审批通过 | 审批流程完成，通过 |
| `REJECTED` | 审批拒绝 | 审批流程完成，拒绝 |
| `RETURNED` | 退回修改 | 退回给申请人修改 |

### 7.2 审批操作枚举

| 操作码 | 操作名 | 说明 |
|--------|--------|------|
| `APPROVE` | 审批通过 | 审批人同意工单 |
| `REJECT` | 审批拒绝 | 审批人拒绝工单 |
| `RETURN` | 退回修改 | 审批人退回工单 |
