按架构图拆模块

每个模块只负责一类问题，避免把所有能力都塞进一个大 Agent。

GsdOrchestrator

主流程编排，负责 sprint / epoch、planner、localizer、executor、evaluator 和状态。

GsdPlanner

把自然语言需求转成 spec / PRD，明确背景、边界和验收基准。

GsdLocalizer

定位可能受影响的文件、类、方法，减少模型盲找。

ContractNegotiator

把需求转成验收条件 AC，让 Evaluator 可以判断完成度。

BuilderPlanRunner

执行前准备目标文件、路径归一、变更契约、策略和上下文。

GateEngine

检查工作区安全状态，避免覆盖用户已有改动。

ExecutionPolicy

决定 solo / auto / team，控制并发和风险。

MutationContract

明确允许写入边界，阻止路径漂移和高风险文件误改。

FreecodeExecutor

唯一代码执行器，负责 XML tool loop、pre-read、写文件、测试和回滚。

TaskStatusReporter

后台状态线程，统一写 PlanState 和 WAL。

Memory Layer

ContextOptimizer、RepoMap、Graphify、LanceDB 共同管理上下文和长期经验。

GsdEvaluator

执行编译、测试和验收，把失败反馈给下一轮。

功能细化

几个最关键的工程机制。

XML Tool Loop

替代不稳定 Patch

通过 read_file、replace、write_file 等 XML 工具让模型一步一步操作，失败后可纠正。

MINIONS

本地 draft + 云端 verify

本地模型降低成本和延迟，云端模型保证关键代码生成质量。

Attempt Rollback

只回滚本轮 touched files

编译失败时不再全仓库清场，避免误伤用户已有改动。

WAL

知道做到哪一步

通过事件日志、PlanState、Checkpoint 和 Reporter 记录长任务进度。

File Partition

避免并发写冲突

同批 worker 不能写同一个文件，边界未知任务自动串行。

Vaccine Memory

避免重复踩坑

连续失败后写入长期记忆，下次相似任务提前召回。
