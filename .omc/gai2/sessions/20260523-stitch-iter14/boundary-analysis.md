# 迭代14 边界推演分析

> 创建时间：2026-05-23
> 对应功能：全局搜索组件（GlobalSearch）+ 报表中心页面（ReportsPage）

---

## 1. 全局搜索组件边界场景

### 1.1 搜索框输入超长文本

| 项目 | 内容 |
|------|------|
| 场景 | 用户在 CommandInput 中输入超过 500 个字符的长文本 |
| 预期风险 | cmdk 内部处理大文本时可能出现性能抖动；UI 展示可能溢出 |
| 当前处理 | cmdk 的 CommandInput 使用原生 input，默认支持长文本输入但无字符限制 |
| 改进建议 | 增加 maxLength 限制（如 100 字符），或使用 text-overflow: ellipsis 裁剪显示 |

### 1.2 快速连续输入

| 项目 | 内容 |
|------|------|
| 场景 | 用户以极快速度连续敲击键盘输入关键词 |
| 预期风险 | cmdk 过滤逻辑频繁触发，可能造成渲染抖动或输入滞后 |
| 当前处理 | cmdk 内部使用 React state 驱动过滤，无 debounce 逻辑 |
| 改进建议 | 在 CommandInput 外层添加 debounce（300ms），降低过滤频率 |

### 1.3 无搜索结果

| 项目 | 内容 |
|------|------|
| 场景 | 输入完全不匹配任何预定义关键词 |
| 预期风险 | 用户可能困惑于为何无结果也不跳转页面 |
| 当前处理 | CommandEmpty 组件显示"未找到结果" |
| 改进建议 | 添加"无匹配结果，按 Enter 前往搜索页面"的 fallback 引导，或显示搜索建议 |

### 1.4 键盘导航越界

| 项目 | 内容 |
|------|------|
| 场景 | 在搜索结果列表最顶部按↑键，或在最底部按↓键 |
| 预期风险 | cmdk 内部循环导航或导航"穿透"到弹窗外元素 |
| 当前处理 | cmdk 的 CommandList 内置循环导航（顶部↑跳到底部，底部↓跳到顶部） |
| 改进建议 | 无额外处理需求，cmdk 原生已处理边界。但需确认无障碍 ARIA 角色正确 |

### 1.5 多次触发 Cmd+K

| 项目 | 内容 |
|------|------|
| 场景 | 弹窗已打开时再次按下 Cmd+K 或点击触发按钮 |
| 预期风险 | 多次 setState 调用导致弹窗闪烁或状态异常 |
| 当前处理 | open 状态由 useState 管理，onOpenChange 回调同步 cmdk 的 Dialog 状态。再次触发时直接 toggle 关闭 |
| 改进建议 | 当前行为合理（toggle 开关），无额外处理需求 |

## 2. 报表中心页面边界场景

### 2.1 报表数据为空

| 项目 | 内容 |
|------|------|
| 场景 | 后端 API 返回空列表（如数据库中无资产数据） |
| 预期风险 | 页面显示空白或加载态卡住 |
| 当前处理 | 使用 EmptyState 组件显示"暂无报表"提示；图表预览也使用 EmptyState 显示"暂无图表数据" |
| 改进建议 | 添加从空数据到引导用户创建资产的链接或提示 |

### 2.2 API 请求失败/超时

| 项目 | 内容 |
|------|------|
| 场景 | 后端服务不可用、网络断开、请求超时（15s） |
| 预期风险 | 页面卡在加载态，组件不展示错误信息 |
| 当前处理 | useQuery 的 isLoading 为 true 时显示 SkeletonCard 骨架屏；失败时不显示错误提示 |
| 改进建议 | 添加 onError 回调，使用 sonner toast 提示"数据加载失败，请重试"；或添加重试按钮 |

### 2.3 所有分类筛选无结果

| 项目 | 内容 |
|------|------|
| 场景 | 用户切换到某个 tab 分类，但该分类下未定义任何报表卡片 |
| 预期风险 | 用户看到"暂无报表"但不知道为什么 |
| 当前处理 | EmptyState 显示"当前分类下暂无可用报表" |
| 改进建议 | 当前处理已充分，用户看到分类名称和空状态可以理解 |

### 2.4 图表数据点过多或过少

| 项目 | 内容 |
|------|------|
| 场景 | 趋势图数据点超过 100 个（如 24 个月期），或只有 1 个数据点 |
| 预期风险 | 图表的 X 轴标签重叠；单数据点折线图/面积图展示不完整 |
| 当前处理 | 无特殊处理，recharts 会根据数据量自动调整刻度 |
| 改进建议 | 过多数据点（>24）时对 X 轴标签进行采样（隔 N 个显示一个）；过少数据点（<3）时提示"数据不足以生成趋势图" |

### 2.5 响应式布局（窄屏/宽屏）

| 项目 | 内容 |
|------|------|
| 场景 | 在 320px 手机屏幕 / 1920px 宽屏显示器上查看 |
| 预期风险 | 窄屏时网格布局可能折行异常或卡片过挤；宽屏时图表区域过宽导致可读性下降 |
| 当前处理 | 网格使用 grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4 响应式断点 |
| 改进建议 | 窄屏时图表预览区调整为全宽并减少 padding；宽屏时限制图表最大宽度 1200px |

## 3. 总结

| 场景 | 严重程度 | 当前覆盖 | 是否需要立即修复 |
|------|---------|---------|----------------|
| 搜索超长文本 | 低 | 部分覆盖 | 否 |
| 快速连续输入 | 中 | 未覆盖 | 否（可后续优化） |
| 无搜索结果 | 中 | 已覆盖 | 否 |
| 键盘导航越界 | 低 | 已覆盖（cmdk） | 否 |
| 多次触发 Cmd+K | 低 | 已覆盖 | 否 |
| 报表数据为空 | 中 | 已覆盖 | 否 |
| API 请求失败 | 高 | 未覆盖 | 是（建议添加错误提示） |
| 分类无结果 | 低 | 已覆盖 | 否 |
| 图表数据异常 | 中 | 未覆盖 | 否（可后续优化） |
| 响应式布局 | 低 | 已覆盖 | 否 |

**建议立即修复项**：API 请求失败的 toast 提示（高严重度，当前未覆盖）。
