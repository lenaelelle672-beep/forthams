# 边界推演 — Iteration 14

> 推演时间：2026-05-23
> 推演范围：全局搜索组件（GlobalSearch）、报表中心页面（ReportsPage）、路由/导航集成

---

## 1. GlobalSearch — Cmd+K 全局搜索组件

### 1.1 空搜索

| 项目 | 内容 |
|------|------|
| **场景** | 用户打开 Cmd+K 面板后不输入任何内容 |
| **预期行为** | 显示所有导航分组和条目（完整列表） |
| **实际行为** | 通过 `CommandEmpty` + `CommandGroup` 渲染 SEARCH_ITEMS 全量列表 |
| **边界值** | 数据源为空数组 → 显示 "未找到匹配的页面" |
| **风险** | 低。全量列表展示正常 |

### 1.2 无匹配结果

| 项目 | 内容 |
|------|------|
| **场景** | 用户输入不存在的页面名称（如 "xyz123"） |
| **预期行为** | 显示空状态提示 "未找到匹配的页面" |
| **实际行为** | cmdk 的 `CommandEmpty` 组件自动处理空结果 |
| **边界值** | 输入与所有 item label 均不匹配 |
| **风险** | 低。cmdk 原生支持 |

### 1.3 超长输入

| 项目 | 内容 |
|------|------|
| **场景** | 用户输入超长字符串（>100 字符） |
| **预期行为** | 输入框正常显示，不触发 UI 异常，搜索过滤正常工作 |
| **实际行为** | cmdk 的 CommandInput 为原生 input，没有长度限制 |
| **风险** | 低。浏览器原生 input 可处理超长文本 |
| **缓解措施** | 如果未来发现性能问题，可添加 `maxLength` prop 或 debounce |

### 1.4 键盘导航

| 项目 | 内容 |
|------|------|
| **场景** | ↑↓ 键选择条目，Enter 确认，Escape 关闭 |
| **预期行为** | cmdk 原生支持键盘导航 |
| **实际行为** | ↑ 选择上一项，↓ 选择下一项，Enter 触发 `onSelect`，Escape 通过 `onOpenChange(false)` 关闭 |
| **边界值** | ↑ 到第一条后继续按 ↑ → cmdk 循环到末尾（默认行为） |
| **风险** | 低。cmdk 键盘导航成熟可靠 |

### 1.5 Escape/Cmd+K 冲突

| 项目 | 内容 |
|------|------|
| **场景** | 面板打开时按 Escape 关闭，面板关闭时按 Cmd+K 打开 |
| **预期行为** | Escape 关闭面板；Cmd+K 在关闭时打开，在打开时关闭（toggle） |
| **实际行为** | `onOpenChange(false)` 关闭；`useEffect` 监听 `metaKey+k` 和 `ctrlKey+k` 做 toggle |
| **边界值** | 面板打开时 Cmd+K → 再次触发 toggle 关闭（可能引起混淆） |
| **风险** | 低。Toggle 行为是常见设计模式 |
| **缓解措施** | 在面板打开时，键盘事件中的 Cmd+K 被 cmdk 捕获，不会到达 document 监听器。如果存在冲突，cmdk 内部处理优先级更高 |

### 1.6 重复打开

| 项目 | 内容 |
|------|------|
| **场景** | 用户快速连续按 Cmd+K 多次 |
| **预期行为** | 面板在该打开时打开，关闭时关闭 |
| **实际行为** | toggle 行为：每次按键切换状态。cmdk 的 Dialog 动画会处理过渡 |
| **风险** | 低 |

### 1.7 Cmd+K 在表单输入中触发

| 项目 | 内容 |
|------|------|
| **场景** | 用户在 input/textarea 中按 Cmd+K 期望执行浏览器功能 |
| **预期行为** | Cmd+K 始终打开搜索面板 |
| **实际行为** | `e.preventDefault()` 阻止所有 Cmd+K 的浏览器默认行为 |
| **风险** | 低。Cmd+K 在浏览器中没有标准默认行为，与大部分 Web 应用的约定一致 |

## 2. ReportsPage — 报表中心页面

### 2.1 空数据（首次使用/无资产）

| 项目 | 内容 |
|------|------|
| **场景** | 系统初始化，无资产数据时打开报表中心 |
| **预期行为** | KPI 卡片显示 0，图表区域显示空状态 |
| **实际行为** | KPI 显示 `0` 或 `0 台`，饼图和柱图显示 `<EmptyState>` 占位 |
| **风险** | 低 |
| **改善方向** | 未来可显示引导提示引导用户录入资产 |

### 2.2 API 失败

| 项目 | 内容 |
|------|------|
| **场景** | `getReportSummary` 或 `getReportByCategory` 请求失败（网络断开/500） |
| **预期行为** | 显示错误占位或重试按钮 |
| **实际行为** | react-query 的 error 状态可通过 `isError` 检测，当前组件未显式处理错误 |
| **风险** | 中。用户界面不会显示错误提示，API 失败时所有值为 `undefined`，KPI 显示 `0` |
| **缓解措施** | 未来迭代可添加 `isError` 判断和错误提示（使用 sonner toast 或内联错误 alert） |

### 2.3 零值展示

| 项目 | 内容 |
|------|------|
| **场景** | 资产总数为 0，但名称/标签依然存在 |
| **预期行为** | 显示 `0`，不显示 `-` 或 `NaN` |
| **实际行为** | 使用 `?? 0` 兜底，所有数值均以 `0` 显示 |
| **风险** | 低 |

### 2.4 后端空列表（by-category 空实现）

| 项目 | 内容 |
|------|------|
| **场景** | `getReportByCategory` 返回空列表（后端 `ReportService.getByCategory()` 当前为占位实现） |
| **预期行为** | 图表区域显示空状态 |
| **实际行为** | `EmptyState` 显示 "暂无分类数据" |
| **风险** | 低。页面正常展示，用户可理解 |
| **后续** | 后端完成后端改造后，数据会自动填充 |

### 2.5 分类名称超长

| 项目 | 内容 |
|------|------|
| **场景** | 资产分类名称超过 20 个字符 |
| **预期行为** | 图表（饼图/柱图）中的标签应截断或自动换行 |
| **实际行为** | recharts 默认会溢出或互相重叠 |
| **风险** | 中。可读性受影响 |
| **缓解措施** | 添加 `tickFormatter` 截断超长名称（当前未实现） |

### 2.6 大量分类数据

| 项目 | 内容 |
|------|------|
| **场景** | 存在 20+ 个资产分类 |
| **预期行为** | 饼图和柱图应正常渲染所有分类 |
| **实际行为** | 8 种颜色循环使用（PIE_COLORS 数组），recharts 对大量数据点性能良好 |
| **风险** | 低 |

## 3. 全局 — 路由/导航集成

### 3.1 路由未配置 404 降级

| 项目 | 内容 |
|------|------|
| **场景** | 用户直接访问 `/reports`（正确配置）或访问 `/reports/xxx`（未定义子路由） |
| **预期行为** | `/reports` → 报表中心页面；`/reports/xxx` → 404 |
| **实际行为** | `/reports` 正确渲染。`/reports/xxx` 触发 `*` 通配符路由，重定向到 `/404` |
| **风险** | 低。已有通配符兜底 |

### 3.2 侧边栏高亮

| 项目 | 内容 |
|------|------|
| **场景** | 用户在 `/reports` 页面时，侧边栏对应项应高亮 |
| **预期行为** | NavLink 的 `isActive` prop 匹配 `/reports` 路径 |
| **实际行为** | NavLink 使用 `to="/reports"`，react-router 自动匹配和设置高亮样式 |
| **风险** | 低 |
| **注意事项** | 如果有子路由如 `/reports/create`，`NavLink` 默认 `end` prop 为 false 时会匹配所有以 `/reports` 开头的路径 |

### 3.3 lazyload 闪烁

| 项目 | 内容 |
|------|------|
| **场景** | 首次访问 `/reports` 时 ReportsPage 组件懒加载 |
| **预期行为** | 显示 `<PageLoader />`（居中旋转动画） |
| **实际行为** | React.lazy + Suspense + S 包装器保证加载态 |
| **风险** | 低 |

### 3.4 全局搜索与侧边栏路由不一致

| 项目 | 内容 |
|------|------|
| **场景** | 用户通过 Cmd+K 搜索到并跳转到某个页面，但侧边栏没有对应入口 |
| **预期行为** | 搜索和侧边栏的导航项应保持同步 |
| **实际行为** | `SEARCH_ITEMS` 与 `NAV_GROUPS` + `NAV_BOTTOM_ITEMS` 手动同步维护 |
| **风险** | 中。未来添加新页面时可能忘记更新 SEARCH_ITEMS |
| **缓解措施** | 考虑将来从 NAV_GROUPS 自动生成搜索数据源，避免重复维护 |

## 4. 推演结论

| 风险等级 | 问题数 | 需关注项 |
|----------|--------|----------|
| 高 | 0 | — |
| 中 | 3 | API 失败未显式处理、分类名称超长、搜索与侧边栏数据源不同步 |
| 低 | 11 | 标准边界覆盖，均有处理或缓解措施 |

**总体评估**：功能在边界条件下表现稳定，没有严重阻塞问题。三个中等问题建议在后续迭代中解决。
