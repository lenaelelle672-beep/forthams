Airtable 方向确认 + 大屏 V2

日常后台走“台账协作 + 企业化约束”；大屏走独立深色科幻驾驶舱。两套视觉共用资产语义，不共用页面风格。

结论先说：Nuxt 不会天然更好看，视觉上真正决定效果的是 DESIGN.md 和组件规范。Nuxt 可能让“新建 Vue/Nuxt 中后台”更快，但对当前 React/Vite 老项目，直接迁移通常更慢、风险更高。

Nuxt 取舍：好看 vs 高效

Decision

推荐：React/Vite + Airtable DESIGN.md + Nuxt checklist

  • 视觉可以一样高级，跟 Nuxt 无直接绑定。
  • 保留现有 React、Vite、Tailwind v4、Radix/shadcn、测试和路由资产。
  • 借鉴 Nuxt Pro 的表格、表单弹窗、API 封装、DTO、权限、导出、i18n、a11y。
视觉上限

可选：Nuxt 新前端并行重写

  • 如果目标是彻底重开新前端，Nuxt UI / shadcn-vue / Pinia 会很顺。
  • 但需要重写路由、状态、组件、测试、权限守卫、API 层。
  • 适合另起 `frontend-nuxt` 试点，不适合直接替换现有 React 主线。
