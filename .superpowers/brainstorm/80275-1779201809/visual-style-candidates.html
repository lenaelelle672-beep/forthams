Nuxt Frontend Design Pro 怎么用

推荐：React/Vite 保留，借鉴 Nuxt checklist

  • 保留当前 React 18、Vite、Tailwind v4、Radix/shadcn 投资。
  • 吸收 API 封装、DTO 对齐、权限、表格、导出、i18n、a11y。
  • 重构成本低，和现有代码迁移风险匹配。

可选：整条前端重开 Nuxt

  • 适合新项目或愿意完整迁移 Vue/Nuxt 的情况。
  • 会涉及路由、状态、组件库、测试、构建链全面替换。
  • 风险高，除非目标是从旧前端切新前端产品线。

我的判断：这次“重构前端设计系统”可以用 Nuxt Pro 的中后台方法论，但不建议把 forthAMS 技术栈切成 Nuxt。
